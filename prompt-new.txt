# Client-side data model & important formulas

(What the app reads from Firestore and how it uses it)

**Firestore collections (read-only for client):**

* `users/{userId}` — username, createdAt, preferences (only for login/profile).&#x20;
* `crimeZones/{streetId}` — fields: `street_name`, `startPoint` (GeoPoint), `endPoint`, `coordinates` (array of GeoPoints), `averageCrimeScore` (ACS numeric), `riskLevel` ("low"/"high"), optionally `geoJsonPolygon`, `lastUpdated`. The client will fetch these GeoJSON polygons for heatmap and for route avoidance/analysis.&#x20;

**Average Crime Score (ACS):**

* Each crime incident has a score (CS): rape=5, homicide=4, murder=4, carnapping vehicle=3, carnapping motorcycle=3, physical injury=2, robbery=2, theft=1. ACS for a street segment = (sum of CSi) / N (N = number of incidents on the segment). High-crime polygons have ACS > 3.0 in the dataset.&#x20;

**Client use of ACS / GeoJSON polygons:**

* For a given Mapbox Directions route, the client will *check intersections* between the route polyline and crime polygons (or compute weighted exposure by sampling points along the route). The client then selects the route with the lowest exposure (ACS) or calls Mapbox Directions with GeoJSON avoid polygons (if supported). These are client-side steps (algorithm described in the step-by-step prompt).&#x20;

# Implementation notes / constraints for Bare React Native + native modules

* **Mapbox & AR are native.** Configure `@rnmapbox/maps` with a Mapbox downloads token for Gradle and an access token at runtime. Ensure `minSdkVersion ≥ 24`, `compileSdkVersion ≥ 34`.
* **Firebase**: Use `@react-native-firebase` modules for app, auth, and firestore. Add `google-services.json` to `android/app` and apply the Google Services Gradle plugin.
* **ViroReact (AR)**: Use `@viro-community/react-viro`. Detect ARCore availability; if unsupported, switch to a 2D fallback.
* **Permissions**: Request camera and location at runtime (`react-native-permissions`). Declare permissions in `AndroidManifest.xml`.
